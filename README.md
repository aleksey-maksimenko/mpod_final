# Распределенный веб-краулер (для товаров интернет-магазинов)

## Описание проекта

Распределенный веб-краулер, предназначенный для параллельного сбора информации о товарах (например, https://poryadok.ru).

Указываем набор URL-адресов, система загружает веб-страницы, извлекает данные (с описанием товара).

По собранным данным можно выполнять поиск, фильтрацию, получать агрегированную статистику.

### Основной пользовательский сценарий

1. Запускаем мастер-узел.
2. Запускаются несколько рабочих узлов.
3. Рабочие узлы регистрируются у мастера.
4. Задаем начальные URL;
5. Мастер помещает URL в очередь задач.
6. Свободные рабочие узлы получают задания.
7. Каждый Worker:
   - загружает HTML;
   - извлекает данные о товарах;
   - находит новые ссылки;
   - возвращает результат мастеру.
8. Мастер:
  - сохраняет результаты и продолжает распределение задач.
9. После завершения обхода доступны:
  - поиск товаров;
  - фильтрация;
  - вычисления статистики.
---

## Архитектура

Архитектура по модели **Master–Worker**.

Мастер-узел отвечает за управление жизненным циклом задач, рабочие узлы выполняют обработку веб-страниц.

Передача заданий и результатов через TCP-соединения.

Внутри каждого воркера обработка в виде асинхронного конвейера на основе **TPL Dataflow**.

Анализ (поиск, вычисления) накопленных данных с использованием **PLINQ**.

---

## Применение параллельных подходов

### 1. Прием входных данных

**BlockingCollection<CrawlerTask>** - потокобезопасная очередь заданий, из которой мастер распределяет URL между рабочими узлами.

---

### 2. Мастер-воркер, взаимодействие между узлами

Мастер хранит очередь URL, принимает подключения рабочих узлов, назначает задания, получает результаты.

- **Master–Worker** - общая координация;
- **ConcurrentDictionary** - хранение информации о зарегистрированных рабочих узлах и активных задачах;
- **TCP-сокеты (TcpListener, TcpClient)** - обмен сообщениями между мастером и рабочими узлами;
- **async/await** - асинхронная передача и прием сообщений.

Рабочие узлы отправляют мастеру heartbeat-сообщения с установленным интервалом.

При отсутствии heartbeat мастер возвращает все задачи недоступного воркера обратно в очередь.

---

### 3. Обработка страниц (парсинг) на рабочем узле

Рабочий узел получает URL, выполняет загрузку HTML, извлечение информации о товарах.

- **async/await** - выполнение HTTP-запросов;
- **TPL Dataflow** - конвейер обработки.

```
Получили URL
      ↓
Загрузили HTML
      ↓
Ответ OK?
      ↓
Парсинг, извлечение данных
      ↓
Формирование результата
```

---

### 4. Анализ собранной информации

Поиск товаров по критериям, фильтрация, вычисление статистик.

- **PLINQ (AsParallel)** — параллельное выполнение запросов.
